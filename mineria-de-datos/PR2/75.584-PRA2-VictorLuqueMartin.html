from sklearn.cluster import KMeans

def elbow_method(X, max_k = 10):
    wss = []
    # Aplicar el método del codo
    for k in range(1, max_k + 1): 
        kmeans = KMeans(n_clusters=k, random_state=42)
        kmeans.fit(X)
        wss.append(kmeans.inertia_)  # Suma de los cuadrados dentro de los clústers
    return wss

3. Ejercicio 2¶
K-Medians es un algoritmo de clustering similar a K-Means, pero en lugar de usar la media, utiliza la mediana de cada dimensión como centro del cluster. Minimiza la suma de distancias absolutas (L1), lo que lo hace más robusto frente a outliers que K-Means, aunque puede ser computacionalmente más costoso. No está disponible en scikit-learn, haremos uso de pyclustering.


















In [9]:






from pyclustering.cluster.kmeans import kmeans
from pyclustering.utils.metric import type_metric, distance_metric
import numpy as np
from sklearn.metrics import silhouette_score, calinski_harabasz_score, davies_bouldin_score
import random

def kmedians(X, k):
    manhattan_metric = distance_metric(type_metric.MANHATTAN)
    initial_centroids = [X[i] for i in random.sample(range(len(X)), k)]
    kmedians_instance = kmeans(data=X, initial_centers=initial_centroids, metric=manhattan_metric)
    kmedians_instance.process()
    clusters = kmedians_instance.get_clusters()
    centers = kmedians_instance.get_centers()
    labels_kmedians = np.empty(len(X), dtype=int)
    for i, cluster in enumerate(clusters):
        labels_kmedians[cluster] = i
    return {
        "labels": labels_kmedians, 
        "centers": centers
    }

X_original_array = X_original.values if hasattr(X_original, 'values') else X_original

4. Ejercicio 3¶
Según la documentación de scikit-learn, no es posible cambiar la métrica de distancia en el algoritmo K-Means. Sin embargo, es posible utilizar modificar la función de distancia haciendo uso de KMeans de pyclustering, que permite utilizar distintas métricas de distancia.

from pyclustering.cluster.kmeans import kmeans
from pyclustering.utils.metric import distance_metric
from pyclustering.cluster.center_initializer import random_center_initializer
from sklearn.metrics import silhouette_score, calinski_harabasz_score, davies_bouldin_score

def kmeans_with_distance(X, k, metric_type=type_metric.EUCLIDEAN):
    metric = distance_metric(metric_type)
    initial_centers = random_center_initializer(X, k, random_state=42).initialize()
    kmeans_instance = kmeans(data=X, initial_centers=initial_centers, metric=metric)
    kmeans_instance.process()
    clusters = kmeans_instance.get_clusters()
    centers = kmeans_instance.get_centers()
    
    labels_kmeans = np.empty(len(X), dtype=int)
    for i, cluster in enumerate(clusters):
        labels_kmeans[cluster] = i
    return {
        "labels": labels_kmeans, 
        "centers": centers
    }

# Convertimos el DataFrame a array
X_original_array = X_original.values if hasattr(X_original, 'values') else X_original
X_procesado_array = X_procesado.values if hasattr(X_procesado, 'values') else X_procesado

metric_type = type_metric.MANHATTAN  # Usamos la métrica Manhattan para KMedians
metric_name = type_metric(metric_type).name
kmeans_custom_metric_original = kmeans_with_distance(X_original_array, k, metric_type)
kmeans_custom_metric_procesado = kmeans_with_distance(X_procesado_array, k, metric_type)


print(f"Evaluando el modelo KMeans con distancia {metric_name} en datos originales (k={k})...")
print("Silhouette Score:", silhouette_score(X_original_array, kmeans_custom_metric_original['labels']))
print("Calinski-Harabasz:", calinski_harabasz_score(X_original_array, kmeans_custom_metric_original['labels']))
print("Davies-Bouldin:", davies_bouldin_score(X_original_array, kmeans_custom_metric_original['labels']))

print(f"\nEvaluando el modelo KMeans con distancia {metric_name} en datos procesados (k={k})...")
print("Silhouette Score:", silhouette_score(X_procesado_array, kmeans_custom_metric_procesado['labels']))
print("Calinski-Harabasz:", calinski_harabasz_score(X_procesado_array, kmeans_custom_metric_procesado['labels']))

Aplicaremos a continuación el algoritmo DBSCAN (Density-Based Spatial Cluestering of Applications with Noise). Los dos hiperparámetros principales de este algoritmo son:


    

  • El hiperparámetro eps define la distancia máxima entre dos puntos para que se consideren parte del mismo cluster. Se puede calcular utilizando gráficos de distancias.
    • 

  • El hiperparámetro min_samples define el número mínimo de puntos necesarios para formar un cluster. Por normal general suele ser $\text{min\_samples} = 2 * \text{dimensiones}$, aunque se puede ajustar según el caso. En nuestro ejemplo será 10, ya que el dataset tiene múltiples dimensiones.
    • 



Hallemos los hipérparámetros para DBSCAN para ambos datasets:

from sklearn.neighbors import NearestNeighbors

X_procesado, y_procesado = df_procesado.drop(columns=['y']), df_procesado['y']

def get_distances(X, min_samples):
    neighbors = NearestNeighbors(n_neighbors=min_samples)
    neighbors.fit(X)
    distances, indices = neighbors.kneighbors(X)
    return np.sort(distances[:, -1])

min_samples = 10  # Regla general para DBSCAN

Conclusiones finales:


Durante el desarrollo de esta práctica se ha llevado a cabo la aplicación de diversas técnicas de minería de datos, abarcando tanto métodos supervisados como no supervisados, utilizando como base el conjunto de datos denominado “Bank Marketing” proveniente del repositorio UCI Machine Learning Repository. El principal propósito ha sido evaluar la efectividad predictiva de diferentes algoritmos.


En cuanto al aprendizaje no supervisado, se ha implementado el algoritmo de agrupamiento K-Means sobre dos versiones del conjunto de datos: una original y otra previamente normalizada. Los resultados indican que la normalización de los datos tiene un impacto significativo en la calidad y formación de los clústeres generados, además de facilitar una interpretación más clara de las agrupaciones obtenidas. No obstante, se ha evidenciado que ciertos factores como el marcado desbalance en las clases y la presencia de variables categóricas con múltiples categorías inciden negativamente en el rendimiento y la precisión de estos modelos no supervisados.


Por otro lado, en el ámbito del aprendizaje supervisado, se han entrenado y evaluado dos modelos principales: un árbol de decisión y un modelo de ensamble basado en AdaBoost. Si bien ambos algoritmos alcanzan niveles aceptables de precisión global (accuracy), métricas complementarias como la precisión (precision), la exhaustividad (recall) y la puntuación F1 (f1-score) ponen de manifiesto que estos modelos presentan dificultades para identificar correctamente la clase minoritaria, que en este contexto representa a los clientes que efectivamente contratan un producto financiero. Este comportamiento es coherente con el fuerte desequilibrio presente en la distribución de clases del dataset, limitando la capacidad de generalización del modelo y su efectividad para detectar casos positivos con relevancia práctica.


En definitiva, esta práctica permite comprender la utilidad de distintos enfoques en aprendizaje automático, así como la necesidad de adaptar las técnicas al contexto del problema. Se destaca que, en problemas reales con datos desbalanceados, el uso de métricas alternativas al accuracy, el preprocesamiento riguroso y la interpretación del dominio del problema son claves para construir modelos útiles y éticamente responsables.
